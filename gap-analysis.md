# Gap Analysis

## Purpose
This file identifies where the sample system does not fully meet the mapped NIST control requirements and tracks what needs to be done next.

## Approach
Each control was reviewed against the sample system and assigned a status based on the current evidence and implementation notes. Each gap is tied to a corresponding risk in the risk register so remediation can be tracked consistently.

## Notes
This gap analysis is a sample portfolio artifact and not a formal assessment report. It is intended to show how control gaps feed into risk tracking and remediation planning.
