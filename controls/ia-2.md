# IA-2 Identification and Authentication

## Control Summary
This control ensures that organizational users are uniquely identified and authenticated before being granted access.

## Implementation Notes
- Require unique user accounts.
- Enforce strong authentication methods.
- Use multi-factor authentication for sensitive access.
- Associate user identity with activity where possible.
- Review authentication settings regularly.

## Evidence
- IAM or authentication policy.
- MFA configuration screenshot or export.
- Sample login or access record.
- User account list or access review record.
